# BamBam — Page Factory Video Engine
## Implementation Plan

---

## The Core Insight

Each Page Factory page has images already published on the WordPress page.
Notion stores only 1 hero image URL per page. The rest are scraped from the live WordPress page.
We don't generate new images — we animate what already exists.

The pipeline is:

```
Notion Page Tracker row (page URL + hero_image_url + content fields)
    ↓
Scrape live WordPress page → extract image URLs (hero always first)
    ↓
[If not enough images] Generate missing ones via Kie.ai API
    ↓
Generate 5-scene voiceover script (Claude API)
    ↓
Generate voiceover audio (Cartesia)
    ↓
Animate images with Ken Burns / pan-zoom (ffmpeg)
    ↓
Assemble: intro → 5 scenes → outro + broll inserts + voiceover
    ↓
[If YouTube API provided] Upload to YouTube with optimized metadata
    ↓
[If YouTube API provided] Return YouTube embed + VideoObject schema → inject into WordPress
    ↓
Write youtube_url + video_status back to Notion Page Tracker
```

If no YouTube API is configured → pipeline runs up to local video output, marks status `ready_to_upload`, and stops. No crash.

---

## Architecture: Generic by Design

Works for ANY niche. All variable content comes from the Notion row. Zero hardcoding.

**What changes per end-client (the dentist, the car shipper, etc.):**
- Notion API key + database ID
- WordPress site URL + credentials
- Brand name + YouTube channel
- Intro/outro clips (we produce these — client does not)
- Cartesia voice selection (optional, has default)

**What we (the developer) own and provide:**
- B-roll library: 100–300 reusable animated clips built from Kie.ai-generated images
- Intro/outro templates: produced using Kie.ai images + ffmpeg, branded per client on demand
- System code and infrastructure

**What the end-client provides (one-time setup):**
- Notion access (already have)
- WordPress access (already have)
- YouTube OAuth credentials (when ready — system works without it)
- Brand name + unique selling point per page in Notion

## Video Structure (5 Scenes, 25–40 seconds)

```
[INTRO 3s]          ← Branded, produced by us
[Scene 1 — Hook 5s] ← Image + caption: city/service keyword hook
[Scene 2 — Problem 5s] ← Image + caption: pain point
[BROLL 2s]          ← Reusable clip from our library
[Scene 3 — Solution 5s] ← Image + caption: brand as solution
[Scene 4 — Trust 5s] ← Image + caption: credibility / social proof
[BROLL 2s]          ← Reusable clip
[Scene 5 — CTA 5s]  ← Image + caption: call to action
[OUTRO 4s]          ← Branded, produced by us
+ VOICEOVER mixed over all scenes
Total: ~36 seconds
```

---

## Script Generation (Claude API)

Each page gets a unique 5-scene script generated by Claude (claude-haiku for cost efficiency).

**Input:** city, state, vertical, brand_name, unique_data_point
**Output:** 5 short lines (~10 words each), one per scene

Example output for `Auto Transport | Phoenix AZ | FastCarShip`:
```
Scene 1 (Hook):    "Shipping a car from Phoenix? Here's what you need to know."
Scene 2 (Problem): "Most transport companies leave you guessing on pickup dates."
Scene 3 (Solution): "FastCarShip guarantees door-to-door pickup within 48 hours."
Scene 4 (Trust):   "Thousands of vehicles delivered. Fully insured. Zero surprises."
Scene 5 (CTA):     "Get your free Phoenix quote today at FastCarShip.com."
```

Cost: ~$0.0005/page with claude-haiku. Effectively free at scale.

---

## Voiceover (Cartesia)

**Primary:** Cartesia API — high-quality, fast, cheap (~$0.003–0.005/page at 60 words)
- Client provides Cartesia API key
- Select voice per client (default: `sonic-english`)

**Fallback (no Cartesia key):** edge-tts — free, robotic but functional for testing

The voiceover is the full 5-scene script read as one continuous audio file, then mixed under the assembled video.

**Note on images:** Only `hero_image_url` is stored in Notion (1 image per page).
Image sourcing priority:
1. Hero image from Notion → always Scene 1
2. Scrape live WordPress page → fill remaining scenes
3. If WordPress yields fewer than the needed count → generate missing images via Kie.ai API using city + vertical as prompt

## YouTube API: Optional, Graceful Skip

The system checks for YouTube credentials at startup:
- **Credentials present:** full pipeline runs → upload → inject into WordPress → update Notion
- **Credentials missing:** pipeline produces local `final_video.mp4`, sets `video_status = ready_to_upload`, stops. No error. Upload can be triggered later once client provides credentials.

YouTube credentials are added via OAuth flow (one-time per client channel). We handle setup.

## B-Roll Library (Developer-Owned)

We produce and maintain the b-roll library using Kie.ai-generated images animated with ffmpeg.
The end-client does not provide b-roll.

**Library structure:**
```
assets/broll/
├── generic/          ← Works for any niche
│   ├── broll_city_01.mp4
│   ├── broll_city_02.mp4
│   └── ...
└── by_vertical/
    ├── auto_transport/
    │   ├── broll_carrier_01.mp4
    │   └── ...
    ├── dental/
    └── cleaning/
```

System selects b-roll by matching `vertical` field → `by_vertical/[vertical]/` → fallback to `generic/`.

**Production:** Each b-roll clip = 1 Kie.ai image + Ken Burns animation + ffmpeg render = ~$0.02/clip.
Build 20–50 clips per vertical. Reused across all pages for that vertical.

## Intro/Outro (Developer-Produced Per Client)

We produce these using Kie.ai images + ffmpeg. End-clients do not make these.

**Per client, we deliver:**
- `intro.mp4` (~3s): Brand logo + brand name animated over a background image
- `outro.mp4` (~4s): CTA text + website URL animated over a background image

Stored in:
```
assets/clients/[client_id]/
├── intro.mp4
└── outro.mp4
```

These are one-time production items. No client effort required.

## Module Details

### `notion_reader.py`
Pulls all rows where `video_status = pending` from the client's Notion Page Tracker DB.
Returns structured dicts with all page fields including image_1 through image_5.

### `page_scraper.py`
Primary image source — always runs.
Scrapes the live WordPress page and extracts up to 5 content images.
Hero image (from Notion) is always placed first, then remaining images follow from the page.

### `image_generator.py`
Fallback — only called when the scraper returns fewer images than needed.
Calls Kie.ai API with a prompt built from `vertical` + `geo_city` to generate the missing images.
Generated images are downloaded and dropped into the same temp folder as scraped images — the assembler never knows the difference.

### `image_downloader.py`
Downloads image URLs to local temp folder. Handles jpg/png/webp.
Skips re-download if file already exists.

### `script_generator.py`
Calls Claude API (claude-haiku-4-5) to generate a 5-scene script.
Returns dict: `{hook, problem, solution, trust, cta}` — one line each.
Full voiceover = all 5 joined. Caption overlays = each line on its matching scene.

### `tts.py`
Primary: Cartesia API → MP3.
Fallback: edge-tts (free) if no `cartesia_api_key` in config.

### `assembler.py`
ffmpeg pipeline:
1. Each image → 5s animated clip (Ken Burns zoom-in or pan, varies per scene)
2. Broll clips selected by vertical
3. Sequence: intro → scene1 → scene2 → broll → scene3 → scene4 → broll → scene5 → outro
4. Caption text burned onto each scene clip
5. All clips concatenated
6. Voiceover mixed in, normalized to -16 LUFS

### `youtube_uploader.py`
Checks if YouTube credentials file exists before attempting upload.
If missing: returns `None`, sets status `ready_to_upload`.
If present: uploads, returns video_id.

### `schema_injector.py`
Generates VideoObject JSON-LD for the page.
Only called if youtube_video_id is not None.

### `wordpress_updater.py`
Injects YouTube embed + schema into the WP page via REST API.
Checks if video already injected (idempotent).
Only called if youtube_video_id is not None.

### `notion_writer.py`
Writes back to Notion Page Tracker:
- `video_status`: `done` / `ready_to_upload` / `failed`
- `youtube_video_id`: filled if uploaded
- `youtube_url`: filled if uploaded

## YouTube API Setup (One-Time Per Client, When Ready)

1. Google Cloud Console → Create project → Enable YouTube Data API v3
2. OAuth 2.0 credentials (Desktop app) → Download `client_secrets.json`
3. Run once:
```python
from google_auth_oauthlib.flow import InstalledAppFlow
flow = InstalledAppFlow.from_client_secrets_file(
    "client_secrets.json",
    scopes=["https://www.googleapis.com/auth/youtube.upload"]
)
creds = flow.run_local_server(port=0)
with open("config/clients/client_abc_youtube.json", "w") as f:
    f.write(creds.to_json())
```
4. Default quota: 6 uploads/day. Request increase → up to 2,000/day (1–5 day approval).

## Scaling Notes

- **YouTube quota:** 6/day default → request increase → up to 2,000/day. At 2K/day: 22K pages = 11 days.
- **Speed:** ~3–5 min/video on 4-core VPS. Add multiprocessing for parallel renders.
- **Cost per video:** Claude script ~$0.0005 + Cartesia voiceover ~$0.004 + compute ~$0.01 = **~$0.015/video**. 22K videos ≈ $330 total.
- **B-roll reuse:** Same 20–50 clips rotate across all pages in a vertical. Viewers never notice at page-level scale.
- **Multi-client:** Each client = own config + own assets folder + own Notion DB. Run separately per client.
